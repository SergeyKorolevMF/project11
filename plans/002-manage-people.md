# План 002: Управление людьми (Репортами)

## Цель (Objective)
Реализовать возможность добавления и просмотра списка "Людей" (Direct Reports), к которым в будущем будут привязываться заметки.

## Шаги (Proposed Steps)

1.  **База данных (Models):**
    *   В `database/models.py` добавить модель `Person`.
    *   Поля: `id` (UUID/Int), `user_id` (FK на User), `name` (Str), `created_at`.
    *   Обновить `database/db.py` (если нужно добавить модель в список модулей, хотя она в том же файле).

2.  **Хендлеры (Handlers):**
    *   Создать новый файл `handlers/people.py`.
    *   Реализовать FSM (машину состояний) для добавления человека:
        *   Команда `/add_person` -> запрос имени.
        *   Ввод имени -> создание записи в БД -> подтверждение.
    *   Реализовать команду `/my_team` (или `/people`):
        *   Выборка всех `Person`, привязанных к текущему `User`.
        *   Вывод списка кнопками (Inline) или текстом.

3.  **Регистрация:**
    *   Подключить новый роутер `people.router` в `bot.py`.

4.  **UX улучшения (Клавиатуры):**
    *   Добавить кнопку "Моя команда" в главное меню (если будем делать постоянную клавиатуру) или просто оставить командами. *Решение: пока командами для скорости.*

## Риски
*   Изменение схемы БД. Так как мы используем `Tortoise.generate_schemas()` в `bot.py`, при старте новые таблицы должны создаться автоматически. (В продакшене нужны миграции, но для MVP на SQLite это сработает, если таблица новая).

## Стратегия отката
*   Удалить модель `Person`.
*   Убрать роутер из `bot.py`.
